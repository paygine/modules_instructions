Плагин Paygine для WooCommerce
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
инструкция по установке

В панели администрирования сайта необходимо выбрать Плагины → Добавить новый и нажать кнопку Добавить новый.

После этого выберите ссылку для загрузки плагина в формате .zip и на открывшейся странице нажмите кнопку Обзор для выбора файла payginewoocommerce.zip. После чего нажмите кнопку Установить.

После установки необходимо активировать плагин.

Для настройки перейдите в WooCommerce → Настройки → Оплата (вкладка) → Paygine

Установите флажок «Включить использование метода платежа Paygine». В поле «Sector ID»
введите идентификатор магазина (можно узнать в личном кабинете на сайте Paygine или в
поддержке). В поле «Пароль» – пароль электронно-цифровой подписи (можно узнать в
личном кабинете на сайте Paygine).
Для того, что проверить сайт в работе, выберите режим Test mode (при этом реальные
платежи не осуществляются) или Production mode (реальная работа).
Для того, чтобы система Paygine уведомляла вас об операциях, которые могут происходить
с задержкой, передайте этот URL для уведомлений службе поддержки Paygine:
http://ВАШСАЙТ.RU/?wc-api=paygine_notify
(замените ВАШСАЙТ.RU на настоящее имя сайта)

FAQ
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Фискальные позиции не передаются
	- убедиться, что установлен новый модуль из гитхаба (https://github.com/paygine/). Если переустановить модуль самостоятельно не удается, обратиться в поддержку с предоставлением доступа к сайту (админка, ftp/ssh)
	- проверить настройку передачи данных на ККТ (при наличии), должно быть включено
	- Код ставки НДС для ККТ от 1 до 6
	- Если ничего не помогло, обратиться в поддержку, предоставив доступ к сайту (админка, ftp/ssh)
 
Не работает оплата - регистрация заказа не прошла
	- убедиться, что установлен новый модуль из гитхаба (https://github.com/paygine/). Если переустановить модуль самостоятельно не удается, обратиться в поддержку с предоставлением доступа к сайту (админка, ftp/ssh)
	- проверить номер сектора
	- проверить пароль
	- убедиться, что модуль работает в соответствующем режиме (боевой/тестовый), пароль для тестового режима по-умолчанию "test"
	- проверить настройку передачи данных на ККТ (при наличии). Если чеки формирует Paygine, значение = 1, если используется фискализация на собственной кассе, значение = 0
	- Код ставки НДС для ККТ от 1 до 6
	- Если ничего не помогло, обратиться в поддержку, предоставив доступ к сайту (админка, ftp/ssh)
	
Не работает оплата - заказ зарегистрирован в ЛК paygine
	- убедиться, что установлен новый модуль из гитхаба (https://github.com/paygine/). Если переустановить модуль самостоятельно не удается, обратиться в поддержку с предоставлением доступа к сайту (админка, ftp/ssh)
	- проверить redirect url в настройках (при наличии)
	- убедиться, что стадийность в настройках корректная
	- Если ничего не помогло, обратиться в поддержку, предоставив доступ к сайту (админка, ftp/ssh)
